# TicketBari Requirement Checklist

- Main layout: navbar, content area and footer included.
- Responsive navbar: logo, home, all tickets, dashboard, login/register and avatar dropdown included.
- Authentication: frontend forms, backend register/login/JWT APIs and BetterAuth server route included.
- Real Google OAuth: BetterAuth Google provider code included; requires real Google credentials to verify.
- Home: hero, six advertised tickets, latest tickets and two extra sections included.
- All Tickets: approved tickets, route search, transport filter, price sorting and pagination included.
- Ticket Details: protected details, countdown, quantity validation and booking modal included.
- User Dashboard: profile, booked tickets and transaction history included.
- Vendor Dashboard: profile, add ticket, added tickets, booking requests and revenue overview included.
- Admin Dashboard: profile, manage tickets, manage users and advertise ticket limit included.
- Backend: Express, MongoDB/Mongoose models, JWT middleware and role-protected routes included.
- Payments: Stripe frontend Payment Element, payment-intent endpoint with demo fallback and booking payment endpoint included.
